# Software Specification: Underground Lottery Management System (LottoTrack)

## 1. บทนำ (Introduction)
LottoTrack เป็น Web Application สำหรับบริหารจัดการการรับส่งหวยใต้ดินแบบครบวงจร พัฒนาขึ้นเพื่อแก้ปัญหาความล่าช้าในการบันทึกข้อมูล (High-Speed Entry) และการจัดการความเสี่ยง (Risk Management) ในระบบเจ้ามือหวย โดยใช้เทคโนโลยีเว็บสมัยใหม่ที่รองรับการใช้งานผ่านมือถือเป็นหลัก (Mobile-First)

## 2. กลุ่มผู้ใช้งาน (User Roles)
*   **Admin (เจ้ามือ):** จัดการเลขอั้น/เลขปิด, กำหนดเพดานรับ (Ceiling), จัดการพนักงาน (Agent), ตรวจสอบรายงานกำไร-ขาดทุน และประมวลผลรางวัล
*   **Agent (คนเดินโพย):** บันทึกรายการแทงของลูกค้า, ตรวจสอบบิล, และดูยอดขายของตนเอง

## 3. ความต้องการด้านฟังก์ชัน (Functional Requirements)

### 3.1 ระบบรับแทงความเร็วสูง (High-Speed Betting Entry)
จุดเด่นของระบบคือการบันทึกข้อมูลแบบ Shortcode และ Virtual Keypad เพื่อลดเวลาในการพิมพ์
*   **Shortcode Parsing:**
    *   `123=100x100` -> 3 บน (100) | 3 โต๊ด (100)
    *   `123=100=` -> 3 บน (100) เท่านั้น
    *   `=25=100` -> 2 ล่าง (100) (เครื่องหมาย `=` นำหน้าหมายถึงเน้นล่าง)
    *   `25=100` -> 2 บน (100) (ค่าเริ่มต้นสำหรับเลข 2 หลัก)
    *   `5=50` -> วิ่งบน (50) (ค่าเริ่มต้นสำหรับเลข 1 หลัก)
*   **Extended Functions:**
    *   `*กลับ123=100` -> กลับเลขทุกประตู (เช่น 123, 132, 213, 231, 312, 321) ตัวละ 100
    *   `*19ประตู5=100` -> เลข 2 ตัวบนที่มี 5 ทั้งหมด (05, 50, 15, 51, ...)
    *   `*รูดหน้า1=100` -> เลข 2 ตัวบนที่ขึ้นต้นด้วย 1 (10, 11, ..., 19)
    *   `*รูดหลัง2=100` -> เลข 2 ตัวบนที่ลงท้ายด้วย 2 (02, 12, ..., 92)
*   **Semicolon Multi-bet:** รองรับการคีย์หลายรายการในบรรทัดเดียว เช่น `123=100;=45=50;*กลับ89=20`

### 3.2 การจัดการความเสี่ยง (Risk Management)
*   **เลขอั้น (Half Pay):** กำหนดเลขที่จ่ายครึ่งราคา (แจ้งเตือนตอนคีย์)
*   **เลขปิด (Closed):** กำหนดเลขที่ไม่รับแทง (บล็อกการบันทึกหรือยืนยันแยก)
*   **เพดานรับ (Ceiling Limits):**
    *   กำหนดวงเงินรับสูงสุดต่อประเภท (เช่น 3 ตัวบน รับได้รวมไม่เกิน 100,000 บาท)
    *   กำหนดวงเงินรายเลข (Per-number Specific Limit)
    *   ระบบตรวจสอบยอดรวม Real-time และแจ้งเตือนเมื่อแตะ 80% ของเพดาน

### 3.3 การประมวลผลรางวัล (Draw Processing)
*   **Input Results:** บันทึกผลรางวัล (3 ตัวบน, 3 ตัวล่าง, 2 ตัวบน, 3 ตัวหน้า)
*   **Auto-Derivation:** ระบบคำนวณผลที่เกี่ยวข้องอัตโนมัติ:
    *   2 ตัวล่าง (จาก 2 หลักสุดท้ายของ 3 ตัวบน)
    *   วิ่งบน (จากหลักหน่วยของ 3 ตัวบน)
    *   วิ่งล่าง (จากหลักหน่วยของ 2 ตัวล่าง)
*   **Auto-Grading:** ปรับสถานะบิลเป็น WIN/LOSE ทันที พร้อมคำนวณเงินรางวัล (Payout) ตามอัตราจ่ายและเลขอั้น

### 3.4 รายงานและการตรวจสอบ (Reporting & Dashboards)
*   **Monitor Dashboard:** แสดงตารางสรุปยอดรับรายเลข เรียงลำดับตามยอดเงิน พร้อมแถบสถานะ (Risk Progress Bar)
*   **Win/Loss Analysis:** สรุปยอดรวมแทง - ยอดจ่ายรางวัล = กำไร/ขาดทุน
*   **Top 10 Numbers:** แสดงรายการเลขยอดนิยมที่มีความเสี่ยงสูงสุด

## 4. โครงสร้างข้อมูล (Internal Data Structure)

### ข้อมูลรายการ (Numbers/Transactions)
| Key | Description | Example |
| :--- | :--- | :--- |
| `id` | Unique ID | `1708...-abcde` |
| `number` | เลขที่แทง | `"123"` |
| `type` | ประเภท (3TOP, 3TOAD, 2BOTTOM, etc.) | `"3TOP"` |
| `amount` | จำนวนเงิน | `100` |
| `status` | สถานะ | `PENDING, WIN, LOSE` |
| `timestamp` | เวลาที่บันทึก | ISO String |

### การตั้งค่าความเสี่ยง (Limit Settings)
*   `defaultOdds`: อัตราจ่ายพื้นฐานรายประเภท
*   `closedNumbers`: รายการเลขอั้น (`HALF`) และเลขปิด (`CLOSED`)
*   `ceilingLimits`: วงเงินรับสูงสุดเริ่มต้นรายประเภท
*   `numberLimits`: วงเงินรับสูงสุดเฉพาะรายเลข
*   `timeSetting`: เวลาเปิด/ปิดรับแทงระบบ

## 5. รายละเอียดทางเทคนิค (Technical Stack)
*   **Frontend Framework:** Next.js (App Router)
*   **UI Library:** MUI (Material UI)
*   **State Management:** React Context API (`NumberContext`, `LimitContext`, `AuthContext`)
*   **Data Persistence:** Browser Local Storage (รองรับการใช้งานแบบ Offline-First ในระดับเบื้องต้น)
*   **Design:** Modern Dark Mode & Responsive (Mobile First)
